## 💬 今日问答

### Q1: Atlas 浏览器的主要功能是什么？收费吗？内核是 Chrome 吗？

**A**:
- **核心功能**:
  - **页面内即时辅助**: 在任何网页直接唤起 ChatGPT，无需切换标签页，自动理解当前页面内容
  - **浏览器记忆**: 记住访问过的网页内容和背景信息，可跨会话引用（如"找到我上周看的招聘信息"）
  - **代理模式（Agent）**: AI 可直接操作浏览器完成任务，如自动搜索、加购物车、下单等

- **收费模式**:
  - 免费版：基础功能可用，但有使用限制
  - Plus ($20/月)：解锁代理模式，更多消息配额
  - Pro ($200/月)：几乎无限使用，优先访问最新模型

- **内核**: **不是 Chrome**，是 OpenAI 自研的独立浏览器内核，专为 AI 集成设计

- **国内使用**: 可用，但需要国际网络连接

### Q2: Atlas 浏览器和 Chrome + ChatGPT 插件有什么区别？

**A**:

| 功能 | 免费 Atlas | Chrome + GPT 插件 |
|------|-----------|-------------------|
| 页面内即时问答 | ✅ 原生集成 | ✅ 基本能做到 |
| 理解当前页面 | ✅ 自动感知 | ⚠️ 需手动分享/复制 |
| 浏览器记忆 | ✅ 跨页面记住上下文 | ❌ 没有 |
| 对话连续性 | ✅ 无缝衔接 | ⚠️ 每次新开标签页 |
| 使用体验 | ✅ 流畅，无割裂感 | ⚠️ 在两个产品间切换 |

**核心差距**:
- Atlas 的 AI 理解你在看什么，直接问"这个多少钱"就行
- 插件需要你先选中文字、点击分享、或复制粘贴

### Q3: 什么是 OCR？有哪些应用场景？

**A**:
- **OCR** = Optical Character Recognition，光学字符识别
- **核心流程**: 检测文字区域 → 识别文字 → 后处理修正
- **典型应用场景**:
  - 门票后台录入（供应商海报提取信息）
  - 发票/凭证处理（财务报销自动识别）
  - 用户上传内容识别（票号、身份证等）
- **常用工具**: PaddleOCR（百度开源，中文强）、Tesseract（Google 开源）、阿里云 OCR、腾讯云 OCR

## 📝 学习笔记

### Boris Cherny 访谈深度整理

**来源**: Lenny's Podcast - "What happens after coding is solved"  
**嘉宾**: Boris Cherny（Claude Code 负责人）  
**时间**: 2026年2月19日

---

#### 核心观点

**1. "编程已被解决"（Coding is Solved）**
> "AI has practically solved coding"

- Claude Code 占 GitHub 公开提交的 **4%**
- 日活用户上个月翻倍
- Spotify 称最好的开发者从12月起就没写过一行代码

**2. "未来所有人都是 Builder"**
> "Anyone can just build software anytime"

- 产品经理可以直接动手做原型
- 设计师可以用 AI 生成可运行代码
- 创业不再需要技术合伙人

**3. "软件工程师职位将消失"**
> "Software engineer is going to start to go away in 2026"

- 不是工程师会失业
- 而是"软件工程师"这个**职位名称**会消失
- 因为**所有人都能写代码**了

**4. 过渡期会很痛苦**
> "It's going to be very disruptive. It's going to be painful for a lot of people."

- 2026年开始职位描述会快速变化
- 需要重新学习如何与 AI 协作
- 一些人会掉队

**5. 给打工人的建议**
> "Don't be scared of them. Experiment with AI tools and learn how they function."

- 现在就开始用
- 理解原理，不只是会用
- 保持开放，拥抱变化

---

#### 类比：印刷术的发明

Boris 引用了古腾堡印刷术：
- **以前**: 只有僧侣能抄写书籍（稀缺技能）
- **印刷术后**: 任何人都能传播思想
- **结果**: 知识爆炸、文艺复兴

**AI 写代码 = 软件开发的印刷术时刻**

---

#### Claude Code 的产品哲学

**1. 不做 IDE，做终端工具**
- 开发者已经在终端里工作
- 融入现有习惯，不强迫改变工作流

**2. 给团队"无限 token"**
- 传统做法：限制 AI 使用量
- Boris 做法：让团队随便用
- 结果：团队找到更多使用场景，产品迭代更快

**3. 少做功能，多做深度**
- 不追求功能多
- 把核心体验做到极致
